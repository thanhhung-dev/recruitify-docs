## **Lưu ý khi assign trong Jira:**

### **1. Cách assign:**
   - Vào **Backlog** hoặc **Board**
   - Click vào từng issue → **Assign** → chọn member
   - Hoặc dùng **Bulk Change** để assign hàng loạt

### **2. Workflow:**
   ```
   To Do → In Progress → Code Review → Done
   ```
   - Người được assign: Update status khi bắt đầu/kết thúc subtask
   - Người 5 (QA): Move task sang "Code Review" sau khi review xong

### **3. Testing & Review:**
   - **Người 5 (QA)** review code của TẤT CẢ subtasks trước khi mark Done
   - **Pull Request Rules:**
     - Mọi PR cần approval từ Người 5
     - Backend PRs cần review từ Người 3 (Senior)
   - **E2E tests** (Subtask 8.1) chỉ chạy sau khi tất cả FE/BE/DB đã hoàn thành

### **4. Critical Dependencies (PHẢI TUÂN THỦ):**
   ```
   ⚠️ BLOCKER:
   Subtask 1.1 (Turborepo) → MUST complete first!

   Dependencies:
   ├─ Subtask 1.1 → Subtask 1.2, 1.3, 1.4, 1.5
   ├─ Subtask 1.3 → Subtask 3.8, 3.1-3.7
   ├─ Subtask 3.1 → Subtask 3.2-3.7
   ├─ Subtask 4.1, 4.2 → Subtask 5.1, 5.2
   ├─ Subtask 4.1 → Subtask 6.1
   ├─ Subtask 5.1 → Subtask 6.2
   ├─ Subtask 6.1, 6.2 → Subtask 7.1
   └─ Subtask 7.1 → Subtask 8.1 (E2E)
   ```

### **5. Daily Standup Format:**
   ```
   Yesterday: "Completed Subtask 3.1 (users table)"
   Today: "Working on Subtask 3.2 (user_profiles table)"
   Blockers: "Waiting for Subtask 1.1 to complete"
   ```

### **6. Definition of Done:**
   Một subtask được coi là **Done** khi:
   - ✅ Code đã được commit và push
   - ✅ Unit tests đã viết và pass
   - ✅ Pull Request đã được approve bởi Người 5 (QA)
   - ✅ Code đã merge vào main branch
   - ✅ CI/CD pipeline pass (nếu có)
   - ✅ Manual testing pass (by QA)
